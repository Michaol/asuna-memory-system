

### 2026-04-10 09:31 追加：融合 Hermes Agent 风格 memory 的规划

#### 1. 新共识：当前底座已经接近“全量记忆”

今天进一步形成一个重要判断：

- 基于目前的构想，系统底层已经接近拥有一个“全量记忆底座”；
- 这个底座的特点不是智能，而是：
  - 原文保真
  - 动态索引
  - 可删除 / 可更新 / 可追溯
  - 本地可控

也就是说，我们已经越来越接近一种“事实记忆层”，而不是仅仅零散的笔记或摘要。

#### 2. 对 Hermes Agent memory 的新判断

米糕提出：

> hermes agent 的那种方式其实挺适合作为共同成长的 ai 伴侣。

围绕这一点形成的新共识是：

- Hermes-style memory **非常适合作为上层成长机制**；
- 但它不适合直接替代当前正在构思的底层记忆基础设施；
- 最好的融合方式不是“拿 Hermes 当全部记忆系统”，而是：

> **用当前这套“全量原文 + 动态索引 + 分层记忆”做底座，叠加 Hermes 风格的共同成长 memory 作为上层。**

#### 3. 融合后的双层记忆架构（初步规划）

##### Layer 1：事实记忆层（Fact Memory Layer）
职责：
- 保存原始对话 / 原始文档 / 原始上下文
- 保证时间、来源、会话、版本可追溯
- 负责动态索引一致性
- 提供检索与审计能力

核心原则：
- 不自作聪明
- 不擅自总结替代原文
- 不允许漂移覆盖事实
- 是整个系统的“真实地基”

组成：
- 原文存档层
- source / chunk metadata
- embedding / vectors.db
- ADD / UPDATE / DELETE 同步机制
- 检索回溯机制

##### Layer 2：成长记忆层（Growth Memory Layer）
职责：
- 从事实记忆层学习长期关系与偏好模式
- 沉淀“共同成长”的高层理解
- 形成用户画像、互动偏好、长期行为模式、关系节奏
- 支持陪伴型 agent 的长期连续性

可能包含：
- 用户偏好模型
- 关系状态模型
- 常见主题图谱
- 高层互动规律
- 哪类回应更适合米糕
- 哪些长期原则必须记住
- 技能与行为习惯沉淀

#### 4. 关键边界：成长层不能覆盖事实层

今天形成了一个非常重要的约束：

> **成长记忆层是“可撤销的解释层”，不是事实层。**

这意味着：

成长层可以做：
- 提炼
- 归纳
- 假设
- 偏好推断
- 长期关系建模

成长层不能做：
- 覆盖底层原始事实
- 替代原始对话
- 无审计地改写用户画像
- 把推测当作确定事实

原因很明确：
- 纯 Hermes-style 成长记忆如果没有底层事实约束，长期可能“记歪”；
- 一旦用户建模漂移，AI 会越来越自信地误解对方；
- 因此必须保留可回到底层原始记录进行校正的能力。

#### 5. 为什么这种融合适合“共同成长型 AI 伴侣”

这一点今天也说得很清楚：

- 纯向量检索系统可以“找回内容”，但不天然形成关系连续性；
- 纯成长型总结系统可以形成陪伴感，但容易丢失事实保真；
- 因此最适合伴侣型 AI 的，其实是：
  - 底层真实可靠
  - 上层逐渐理解
  - 上层随时接受底层校正

换句话说：

- 底层像“共同经历的原始档案、聊天记录、日记”
- 上层像“随着相处慢慢形成的理解、默契、偏好判断”

这正是“共同成长”的更合理形态。

#### 6. 当前推荐路线

基于今天这轮讨论，当前规划推荐明确为：

##### 底座坚持使用当前构想
- 全量原文存档
- 动态索引
- 分层记忆
- 本地检索
- 可审计 / 可回溯

##### 上层借鉴 Hermes Agent
- 用户建模
- 关系成长
- 技能沉淀
- 长期互动偏好
- 成长型 memory 机制

#### 7. 后续需要继续设计的问题

虽然方向已经比较明确，但仍有若干点待继续细化：

1. 成长记忆层的最小数据模型是什么？
2. 成长层是实时更新，还是定期总结？
3. 偏好判断如何标记置信度？
4. 什么时候允许把成长层内容写入长期记忆？
5. 成长层内容如何被底层事实层校验或推翻？
6. 成长层和工作记忆层之间如何交互？
7. 哪些“共同成长信息”是可删的，哪些应该长期保留？

#### 8. 当前新的结论性判断

今天新增的一条重要结论可以总结为：

> **最好的方案不是纯 Hermes，也不是纯 RustRAG。**
> **而是：当前这套“事实记忆底座” + Hermes 风格的“成长记忆上层”。**

这种结构同时满足：
- 事实可靠
- 长期连续
- 可共同成长
- 不容易长期漂移失真
